# Project Tags

This file tracks the approved tag set for project entries.

## Origin Tags

- `university`: Projects created through university coursework or academic team work.
- `personal`: Independent projects created outside formal academic or professional work.
- `professional`: Professional client, job, or production work.

## Project-Type Tags

- `application`: End-user applications, products, or interfaces.
- `developer-tool`: Tools, utilities, libraries, or infrastructure meant to support developers or technical workflows.
- `game`: Play-focused interactive projects, prototypes, or game collections.
- `simulation`: Systems that model real-world processes, workflows, or hardware behavior.
- `systems`: Lower-level systems work, protocols, runtimes, serialization, networking, signal processing, or infrastructure-heavy engineering.
- `ai`: Projects where AI is a central part of the product or technical implementation.

## Notes

- Use origin tags alongside one or more project-type tags.
- Prefer tags that describe the project's identity, not just its stack.
- Avoid broad implementation-only tags like `web`, `data`, or `library` unless the tag system is revised again later.
